## ğŸ”„ Request-Response Flow

### Chat Request Flow
```
1. User types message in browser
2. script.js captures input
3. Fetch POST request to /chat
4. FastAPI receives request
5. Routes handler in routes.py
6. Create ChatRequest object
7. Call dialog_manager.process_message()
8. intent_model.predict() - classify intent
9. entity_parser.extract_all() - get entities
10. Generate contextual response
11. Return ChatResponse JSON
12. script.js displays response in chat
13. Updates conversation state
```

### Reservation Request Flow
```
1. User submits reservation form
2. script.js validates form
3. Call /availability endpoint first
4. Check if tables available
5. If available, POST to /reservation
6. routes.py handler:
   - Find/create customer
   - Check double booking (availability.py)
   - Select best table
   - Create Reservation record
   - Generate confirmation ID
   - Save to database
7. Return ReservationResponse
8. script.js displays confirmation
9. Shows confirmation ID
```

### Availability Check Flow
```
1. User asks "Check availability"
2. Dialog manager fills slots
3. POST /availability
4. availability.py gets_available_tables()
5. Check operating hours
6. Validate date (future only)
7. Query tables with capacity >= guests
8. Check for overlapping reservations
9. Return available tables
10. If none, find_alternative_slots()
11. Suggest closest times
12. Return AvailabilityResponse
13. Display to user
```
